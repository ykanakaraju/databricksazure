
 Lab 5: Setup Databricks CLI in a Python Virtual Environment on Windows 10
 --------------------------------------------------------------------------
 
 1. Open 'Anaconda Command Prompt' and install python dev. environement as described below:
 
 2. Create a working directory (DatabricksProjects) for all your Databricks projects
 
 	mkdir E:\Databricks\Projects
	cd E:\Databricks\Projects
		
 3. Create a python virtual environment in your project working directory.
 
	python -m venv dbcli-venv			
	-> This creates a directory called 'databricks-venv' with all basic python packages in the current folder.
	
 4. Activate the v-env
 
	cd .\dbcli-venv\Scripts
	activate      (this will activate the virtual environement)
	
	NOTE: To deactivate the v-env, run 'deacticate' in the same folder (i.e databricks-venv\Scripts)

	(source <venv-dir>/bin/activate  -> equivalent command in Linux)
	
 5. Install Databricks CLI.
 	cd E:\Databricks\Projects
	pip install databricks-cli
	databricks -v     		// check databricks cli version to validate
		
 6. Login to Azure portal, create and launch a new Databricks Workspace 
	 (Refer Lab 2 for detailed steps)

	Workspace name: cts_demo_adb_ws
	Region: East US
	Pricing Tier: Standard
	
	-> Wait until the workspace is deployed, and then launch the workspace.
